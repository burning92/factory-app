# 이카운트 재고 연동 1단계: 데이터 구조 확정

- **목적:** RAW 재고현황 시트와 품목마스터를 품목코드 기준으로 안정적으로 조인할 수 있는 구조 점검 및 확정
- **범위:** 구조·검증·제외 규칙·스키마 초안만 정리. Supabase/앱 구현은 다음 단계.

---

## 1. 1단계 구조 점검 체크리스트

| # | 항목 | 확인 |
|---|------|------|
| 1 | 품목마스터 **코드**가 RAW 시트의 품목 식별자와 1:1 대응 가능한지 | □ |
| 2 | RAW 시트에 **품목코드** 또는 마스터와 매칭 가능한 컬럼이 있는지 | □ |
| 3 | 조인 키 후보(코드 vs 품목명) 중 **코드**를 우선 사용할지 | □ |
| 4 | 재고 단위(박스/개/g)가 마스터의 1박스(g)·1개(g)와 일관되는지 | □ |
| 5 | RAW의 시리얼/로트No.가 LOT 단위 재고 관리에 사용할 필드인지 | □ |
| 6 | 사용여부(Y/N)로 앱 노출/미노출을 제어할지 | □ |

---

## 2. 재고_품목마스터 – 필수 컬럼 구조 및 검증 규칙

### 2.1 컬럼 구조 (현재 기준)

| 컬럼명 | 용도 | 필수 여부 | 비고 |
|--------|------|-----------|------|
| **코드** | 품목 고유 식별자, RAW와 조인 키 | **필수** | 빈칸·중복 불가 |
| 품목명 | 표시명, RAW와 교차 검증용 | **필수** | 빈칸 불가 |
| 재고분류 | 원재료/부재료/반제품 등 구분 | **필수** | 허용값 목록 고정 |
| 카테고리 | 세부 분류(선택) | 선택 | 빈칸 허용 |
| 1박스(g) | 박스 단위 중량 | 조건부 | 박스 단위 사용 시 필수 |
| 1개(g) | 낱개 단위 중량 | 조건부 | g 단위만 있으면 0 가능 |
| 사용여부 | 앱 노출 여부 | **필수** | Y / N 만 허용 |
| 비고 | 메모 | 선택 | 빈칸 허용 |

### 2.2 필수 검증 규칙 (품목마스터)

- **코드**
  - 빈칸(공백만 포함) 금지.
  - 시트 내 **중복 금지**. (동일 코드 2건 이상이면 오류로 처리하거나 첫 행만 사용하도록 규칙 명시)
- **품목명**
  - 빈칸(공백만) 금지.
  - 합계/소계용 행은 품목명 규칙으로 구분(예: 끝이 `계`인 행은 마스터 행이 아님 → 아래 제외 규칙으로 처리 가능).
- **재고분류**
  - 빈칸 금지.
  - 허용값만 허용. (예: `원재료`, `부재료`, `반제품` – 실제 시트에 쓰인 값으로 허용 목록 확정 필요)
- **1박스(g), 1개(g)**
  - 숫자 또는 빈칸. 빈칸이면 0으로 간주할지 규칙 명시 (예: 빈칸 → 0).
  - 음수 불가.
  - 1박스(g)=0 이고 1개(g)=0 이면 “g 전용” 품목으로 해석할지 사전 정의.
- **사용여부**
  - **Y** 또는 **N** 만 허용 (대소문자 통일 규칙 권장: 대문자로 정규화).
  - 빈칸은 N으로 간주할지, 오류로 처리할지 명시 (권장: 빈칸 → N).
- **카테고리, 비고**
  - 선택 항목. 빈칸 허용.
  - 앞단 검증에서는 길이 제한만 둘 수 있음 (예: 200자 이내).

### 2.3 품목마스터 “표시 기준”

- 앱/다음 단계에서 **표시용 품목명**은 **품목마스터의 품목명**을 기준으로 사용.
- RAW의 품목명은 **raw_item_name** 등으로 보관해 두고, 마스터와 불일치 시 경고/로깅만 할지, 조인 실패로 볼지 규칙 확정.

---

## 3. RAW 시트 – 읽어야 할 컬럼 구조

**파일:** 이카운트 RAW 데이터 베이스  
**시트명:** 시리얼/로트No. 재고현황->원재료,부재료,반제품

### 3.1 실제로 읽어야 할 컬럼 (후보)

| 순서 | 컬럼 후보명 | 용도 | 필수 여부 |
|------|-------------|------|-----------|
| 1 | **품목코드** (또는 코드) | 마스터와 조인 키 | **필수** |
| 2 | 품목명 | RAW 기준 품목명, 검증/로깅용 | 권장 |
| 3 | **시리얼/로트No.** (또는 LOT 번호) | LOT 단위 재고 식별 | **필수** |
| 4 | **재고수량** | 해당 LOT의 수량 | **필수** |

- 실제 시트의 **컬럼 헤더 이름**이 위와 다를 수 있으므로, 연동 전에 시트 1행 기준으로 매핑표 작성 권장.  
  예: 시트 헤더 `제품코드` → 품목코드, `로트번호` → 시리얼/로트No.
- 재고 단위(개, 박스, g 등)가 RAW에 별도 컬럼으로 있으면, 그 컬럼도 “읽어야 할 컬럼”에 추가하고, 마스터의 1박스(g)·1개(g)와 조합해 g 환산 규칙을 명시.

### 3.2 조인 규칙

- **조인 키:** RAW의 **품목코드** = 마스터의 **코드**.
- 조인 결과 마스터 행이 없으면: 해당 RAW 행은 “미매칭”으로 분류하고, 동기화 시 제외하거나 별도 큐로 둘지 규칙 확정.

---

## 4. 동기화 시 제외해야 할 행 규칙

### 4.1 품목마스터 시트

- **코드**가 비어 있거나 공백만 있는 행 → 제외 (또는 오류 로그 후 제외).
- **코드** 중복 행 → 첫 행만 사용하고 나머지 제외, 또는 오류로 처리.
- **품목명**이 비어 있거나 공백만 있는 행 → 제외.
- **재고분류**가 비어 있는 행 → 제외.
- **사용여부**가 Y/N이 아닌 행 → 제외 또는 N으로 정규화 후 포함 (규칙 명시).
- 품목명 끝이 **`계`** 인 행(합계행) → 제외.

### 4.2 RAW 재고현황 시트

- **품목코드**(또는 매핑된 코드 컬럼)가 비어 있거나 공백만 있는 행 → 제외.
- **시리얼/로트No.**가 비어 있거나 공백만 있는 행 → 제외 (LOT 없으면 재고 행으로 사용 불가).
- **재고수량**이 비어 있거나 숫자로 파싱 불가한 행 → 제외.
- 재고수량이 **음수**인 행 → 제외 또는 별도 규칙(반품/조정 등)으로 정의 후 처리.
- 품목명 끝이 **`계`** 인 행(합계/소계) → 제외.
- **헤더 행**은 데이터 행으로 읽지 않도록 “첫 N행 스킵” 또는 “헤더 행 번호” 명시.

---

## 5. 다음 단계 입력 스키마 초안 (Supabase 등으로 넘길 때 사용)

아래는 “동기화 결과” 또는 “앱이 사용할 재고/품목 뷰”를 위한 **데이터 계약** 초안. 실제 테이블명·타입은 Supabase 설계 시 조정.

### 5.1 품목 마스터 계약 (item_master)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| item_code | string (PK) | Y | 품목 고유 코드 (마스터 코드) |
| item_name | string | Y | 표시용 품목명 (마스터 품목명) |
| inventory_type | string | Y | 재고분류: 원재료 / 부재료 / 반제품 등 |
| category | string | N | 카테고리 |
| box_weight_g | number | N | 1박스(g), 없으면 0 |
| unit_weight_g | number | N | 1개(g), 없으면 0 |
| is_active | boolean | Y | 사용여부 (Y→true, N→false) |
| note | string | N | 비고 |

### 5.2 RAW 재고 행 계약 (inventory_raw 또는 동기화 중간 스키마)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| item_code | string | Y | 품목코드 (마스터와 조인 키) |
| raw_item_name | string | N | RAW 시트의 품목명 (검증/로깅용) |
| lot_no | string | Y | 시리얼/로트No. |
| qty | number | Y | 재고수량 (해당 LOT) |

### 5.3 조인 결과 / 앱용 뷰 계약 (재고 현황 1행)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| item_code | string | Y | 품목코드 |
| display_item_name | string | Y | 표시용 품목명 (마스터 item_name) |
| raw_item_name | string | N | RAW 품목명 |
| inventory_type | string | Y | 재고분류 |
| category | string | N | 카테고리 |
| lot_no | string | Y | 시리얼/로트No. |
| qty | number | Y | 재고수량 |
| box_weight_g | number | N | 1박스(g) |
| unit_weight_g | number | N | 1개(g) |
| is_active | boolean | Y | 사용여부 |

- **display_item_name**은 항상 마스터의 품목명을 사용해, 시트별 표기 차이로 인한 흔들림을 막는다.

---

## 6. 요약

- **조인 키:** 마스터 **코드** = RAW **품목코드**.
- **마스터 검증:** 코드/품목명/재고분류 필수, 코드 중복·빈칸 금지, 사용여부 Y/N, 1박스(g)/1개(g) 비음수.
- **RAW 읽기:** 품목코드, 품목명, 시리얼/로트No., 재고수량 위주로 읽고, 실제 시트 헤더와 매핑표 확정 필요.
- **제외:** 코드/품목코드/LOT/재고수량 빈칸, 합계행(품목명 끝 `계`), (선택) 재고수량 음수.
- **다음 단계:** 위 5.1~5.3 스키마를 기준으로 Supabase 테이블·뷰 설계 및 연동 로직 설계.
